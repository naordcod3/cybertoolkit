Basic Concepts

Users

Think of users like different people who can log into the computer. Each user has their own "account".

Groups

Groups are like teams. Users can be members of different groups, just like people can be on different teams.

Permissions

Permissions are like keys that control what users and groups can do with files and folders.

Permission Types

Read (r)

Like being able to:

  • See what's inside a folder
  • Open and read a file
  • Copy a file

Write (w)

Like being able to:

  • Create new files
  • Modify files
  • Delete files

Execute (x)

Like being able to:

  • Enter a folder
  • Run a program
  • Use a script

Permission Numbers

Number Permission Meaning
7 rwx Can read, write, and execute
6 rw- Can read and write
5 r-x Can read and execute
4 r-- Can only read
0 --- No permissions

Common Examples

chmod 770 folder

7 (rwx) - Owner can do everything
7 (rwx) - Group can do everything
0 (---) - Others can do nothing

chmod 755 folder

7 (rwx) - Owner can do everything
5 (r-x) - Group can read and execute
5 (r-x) - Others can read and execute

chmod 644 file

6 (rw-) - Owner can read and write
4 (r--) - Group can only read
4 (r--) - Others can only read

Basic Storage Concepts

What is a Mount Point?

A mount point is like a door that gives you access to a storage device. When you mount a device, you're connecting it to a specific directory in your file system.

Example: When you plug in a USB drive, it might be mounted at /media/usb, making its contents accessible through that directory.

What are Partitions?

Partitions are like separate rooms on a hard drive. Each partition can have its own filesystem and be used for different purposes.

Common practice: Separate partitions for / (root), /home (user files), and swap (virtual memory)

Understanding File Systems

A file system is like an organizational system for your storage. It determines how data is stored and retrieved.

Key Features:

  • File permissions
  • File fragmentation handling
  • Journaling (crash recovery)
  • Maximum file sizes

Device Naming Logic

Linux uses a systematic way to name devices:

  • sd = SATA/SCSI drives
  • hd = (older) IDE drives
  • nvme = NVMe SSDs
  • a, b, c... = Drive order
  • 1, 2, 3... = Partition numbers

Common Scenarios

Mounting a USB Drive

sudo mount /dev/sdb1 /media/usb

Checking Disk Space

df -h /home

Creating a New Partition

sudo fdisk /dev/sdb

Best Practices

  • ✓ Always unmount devices before removing them
  • ✓ Regularly check disk space usage
  • ✓ Keep backups before partition changes
  • ✓ Use UUIDs in /etc/fstab for stability

Tip: Use blkid to get device UUIDs

Basic APT Concepts

What is APT?

APT (Advanced Package Tool) is a package management system used in Debian and its derivatives like Ubuntu. It simplifies the process of installing, updating, and removing software.

Repositories

Software sources that contain packages. They're like app stores for Linux, containing thousands of pre-configured software packages ready to install.

Dependencies

Additional packages required by software to function. APT automatically handles these, installing or updating them as needed.

Package Operations

Installation

sudo apt install package_name

Basic installation

sudo apt install package1 package2

Install multiple packages

sudo apt install ./package.deb

Install from .deb file

sudo apt install -y package_name

Install without confirmation

Removal

sudo apt remove package_name

Remove package (keeps config)

sudo apt purge package_name

Remove package and config files

sudo apt autoremove

Remove unused dependencies

sudo apt clean

Remove downloaded package files

Updates

sudo apt update

Update package list

sudo apt upgrade

Upgrade installed packages

sudo apt full-upgrade

Upgrade with dependency handling

sudo apt dist-upgrade

Upgrade distribution

Download

Download package file only

apt source package_name

Download source code

sudo apt install --download-only

Download without installing

sudo apt --reinstall install

Download and reinstall

Basic Penetration Testing Concepts

What is Pentesting?

Penetration testing is the practice of testing a computer system, network or application to find security vulnerabilities that an attacker could exploit.

Methodology

A structured approach including reconnaissance, scanning, gaining access, maintaining access, and covering tracks.

Legal Considerations

Always obtain explicit permission before testing. Unauthorized testing can result in legal consequences.
