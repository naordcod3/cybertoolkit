Overview

Set up a Domain Controller (`Gil.com`) and configure Active Directory with proper user management and security policies.

Instructions

  • Open PowerShell as Administrator
  • Install Active Directory Services using:
    Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools
  • To set up the domain controller, run:
    Install-ADDSForest -DomainName "Gil.com" -DomainNetBiosName "GIL" -InstallDns:$true -Force
  • Create OUs for each department
  • Add users to appropriate OUs
  • Create and configure security groups
  • Join client machines to domain
  • Block PowerShell for Marketing
  • Block CMD for Finance 
# Install Active Directory Services
Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools

# Create a New Forest and Set Up the Domain
Install-ADDSForest -DomainName "Gil.com" -DomainNetBiosName "GIL" -InstallDns:$true -Force

# Create OUs
New-ADOrganizationalUnit -Name "Managers"
New-ADOrganizationalUnit -Name "HR"
New-ADOrganizationalUnit -Name "Finance"
New-ADOrganizationalUnit -Name "Marketing"
New-ADOrganizationalUnit -Name "Development"
New-ADOrganizationalUnit -Name "IT"

# Create Users in Each Department (Example for HR)
for ($i=1; $i -le 5; $i++) {
    New-ADUser -Name "HRUser$i" -AccountPassword (ConvertTo-SecureString "Pass123!" -AsPlainText -Force) -Enabled $true -Path "OU=HR,DC=Gil,DC=com"
}

# Create Security Groups and Add Users (Example for HR)
New-ADGroup -Name "HRGroup" -GroupCategory Security -GroupScope Global -Path "OU=HR,DC=Gil,DC=com"
Get-ADUser -Filter {Name -like "HRUser*"} | ForEach-Object {Add-ADGroupMember -Identity "HRGroup" -Members $_.SamAccountName}

# Join Windows 10 Clients to the Domain (Run on the Client Machines)
Add-Computer -DomainName "Gil.com" -Credential Gil\Admin -Restart

# Block CMD for Finance

Overview

Set up a Windows Server environment with Active Directory, user groups, and storage management.

  • Create 5 virtual machines (3 Windows 10, 2 Windows Server 2022 without GUI).
  • Install VMware Tools and take a snapshot for each machine.
  • Set up networking so all VMs can communicate within `172.16.0.0/16`.
  • Promote the `172.16.0.100` server as a Domain Controller for `gil.com`.
  • Join all machines, including servers, to the domain.
  • Rename computers as specified in the network diagram.
  • Create Organizational Units (OUs) in Active Directory.
  • Create 6 departments and add 5 users per department.
  • Set up a Storage Server (`172.16.0.110`) with six new HDDs.
  • Partition disks into 4x `2.5GB` and 2x `10GB` partitions.
  • Assign and configure shared storage with AD security groups.
  • Verify that each department's users can access only their allocated storage.

Implementation Steps

# Step 1: Create Virtual Machines
# - 3 Windows 10 VMs
# - 2 Windows Server 2022 VMs (No GUI)
# - Install VMware Tools and take snapshots

# Step 2: Configure Networking (172.16.0.0/16)
# - Assign IP addresses based on topology
# - Ensure all machines can communicate

# Step 3: Promote the Domain Controller (172.16.0.100)
Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools
Install-ADDSForest -DomainName "gil.com" -DomainNetBiosName "GIL" -InstallDns:$true -Force

# Step 4: Join All Machines to the Domain
Add-Computer -DomainName "gil.com" -Credential gil\Admin -Restart

# Step 5: Create Organizational Units and Users in AD
New-ADOrganizationalUnit -Name "Managers"
New-ADOrganizationalUnit -Name "Finance"
New-ADOrganizationalUnit -Name "HR"
New-ADOrganizationalUnit -Name "Marketing"
New-ADOrganizationalUnit -Name "IT"
New-ADOrganizationalUnit -Name "Development"

for ($i=1; $i -le 5; $i++) {
    New-ADUser -Name "HRUser$i" -AccountPassword (ConvertTo-SecureString "Pass123!" -AsPlainText -Force) -Enabled $true -Path "OU=HR,DC=gil,DC=com"
}

# Step 6: Configure Storage Server (172.16.0.110)
# - Add 6 HDDs (5GB each)
# - Create 6 partitions (4x 2.5GB, 2x 10GB)
New-VHD -Path "C:\Storage\Disk1.vhdx" -SizeBytes 5GB -Dynamic
New-Partition -DiskNumber 1 -UseMaximumSize -AssignDriveLetter | Format-Volume -FileSystem NTFS -NewFileSystemLabel "Storage1"

# Step 7: Set Up Security Groups & Permissions in AD
New-ADGroup -Name "HRGroup" -GroupCategory Security -GroupScope Global -Path "OU=HR,DC=gil,DC=com"
Get-ADUser -Filter {Name -like "HRUser*"} | ForEach-Object {Add-ADGroupMember -Identity "HRGroup" -Members $_.SamAccountName}

# Step 8: Share Storage Based on Department
New-SmbShare -Name "HR_Share" -Path "C:\HR" -FullAccess "HRGroup"

# Step 9: Verify Access
# - Log in with a user from each department
# - Ensure users can only access their assigned shared folder

# Error Handling Section:

# If you see "NTFS signature is missing" or "Invalid argument":
# This means you're trying to mount a non-NTFS partition as NTFS
# Solution: Make sure to format with ext4 first:
sudo mkfs.ext4 /dev/sdXY  # Replace X with disk letter, Y with partition number

# If you see "Mount is denied because volume is already exclusively opened":
# 1. Check what's using the partition:
sudo fuser -m /dev/sdXY
# 2. Check if it's already mounted:
mount | grep sdXY
# 3. Unmount if needed:
sudo umount /dev/sdXY

# If you see "systemd still uses the old version":
sudo systemctl daemon-reload
sudo mount -a

# If mount fails after editing fstab:
# 1. Check your fstab syntax:
cat /etc/fstab
# 2. Restore backup if needed:
sudo cp /etc/fstab.backup /etc/fstab
# 3. Try mounting manually to test:
sudo mount /dev/sdXY /mnt/mount_point

# General troubleshooting:
# 1. Check partition exists:
sudo fdisk -l
# 2. Check filesystem type:
sudo blkid /dev/sdXY
# 3. Check mount point exists:
ls -l /mnt
# 4. Check mount point is empty:
ls -la /mnt/mount_point

Overview

Merge four 2.5GB NTFS partitions into one 10GB drive using Linux MD (Multiple Devices) feature.

Step by Step Instructions

# Step 1: Install required tools
sudo apt update
sudo apt install mdadm ntfs-3g -y

# Step 2: Unmount the partitions
sudo umount /mnt/Disk1
sudo umount /mnt/Disk2
sudo umount /mnt/Disk3
sudo umount /mnt/Disk4

# Step 3: Create the MD array (RAID 0 for merging)
sudo mdadm --create /dev/md0 --level=0 --raid-devices=4 /dev/sdb1 /dev/sdb2 /dev/sdc1 /dev/sdc2

# Step 4: Format the merged drive as NTFS
sudo mkfs.ntfs -F -f /dev/md0

# Step 5: Create mount point
sudo mkdir -p /mnt/merged_drive

# Step 6: Mount the merged drive
sudo mount -t ntfs /dev/md0 /mnt/merged_drive

# Step 7: Add to fstab for persistence
# First get UUID
sudo blkid /dev/md0

# Add to fstab
sudo nano /etc/fstab
# Add this line (replace xxx with actual UUID):
UUID=xxx  /mnt/merged_drive  ntfs  defaults  0  0

# Step 8: Save MD configuration
sudo mdadm --detail --scan | sudo tee -a /etc/mdadm/mdadm.conf

# Step 9: Update initramfs
sudo update-initramfs -u

# Step 10: Apply changes
sudo systemctl daemon-reload
sudo mount -a

# Step 11: Verify
df -h /mnt/merged_drive  # Should show ~10GB
cat /proc/mdstat         # Check RAID status
sudo mdadm --detail /dev/md0

# If you need to start over:
# sudo umount /mnt/merged_drive
# sudo mdadm --stop /dev/md0
# sudo mdadm --remove /dev/md0
# sudo mdadm --zero-superblock /dev/sdb1 /dev/sdb2 /dev/sdc1 /dev/sdc2

Important Notes

  • This will create a RAID 0 array (striping)
  • All data on the partitions will be lost during merge
  • The merged drive will be approximately 10GB (4 × 2.5GB)
  • If any partition fails, all data will be lost (RAID 0 characteristic)
